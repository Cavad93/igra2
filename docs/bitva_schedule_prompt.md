КРИТИЧЕСКИ ВАЖНО — ВЕТКА
════════════════════════════════════════

Ты работаешь ТОЛЬКО в одной ветке:
  claude/historical-population-distribution-euy06

ПЕРВАЯ команда которую ты выполняешь — всегда:
  git checkout claude/historical-population-distribution-euy06

НИКОГДА не создавай новые ветки.
НИКОГДА не используй git checkout -b
НИКОГДА не используй worktree
НИКОГДА не используй git stash если не уверен что делаешь

Все коммиты и пуши идут ТОЛЬКО в:
  git push -u origin claude/historical-population-distribution-euy06

Если push отклонён (rejected) — сделай:
  git pull --rebase origin claude/historical-population-distribution-euy06
  git push -u origin claude/historical-population-distribution-euy06

════════════════════════════════════════
КОНТЕКСТ ИГРЫ — ЧИТАЙ ВНИМАТЕЛЬНО
════════════════════════════════════════

Это браузерная стратегия на vanilla JS + Canvas 2D, без фреймворков.
Рабочая директория: /home/user/igra2

ВАЖНЫЕ КОНСТАНТЫ ИГРЫ:
- 1 ход = 1 месяц игрового времени
- 1 единица ресурса = 1 кг
- Армии: поля infantry, cavalry, archers (число солдат)
- Экономика: treasury (золото), income (доход/ход)

Ключевые файлы:
- index.html          — главный файл, сюда подключаются все скрипты
- engine/armies.js    — движок армий, resolveBattle()
- engine/battle.js    — логика боя
- engine/combat.js    — формулы урона
- engine/turn.js      — главный игровой цикл
- ui/battle_result.js — анимация результата боя (showBattleResult)
- ui/map_armies.js    — отображение армий на карте
- GAME_STATE          — глобальный объект состояния игры

Файлы которые СОЗДАЮТСЯ в рамках этого плана:
- engine/tactical_battle.js  — логика тактического боя
- ui/tactical_map.js         — Canvas-рендер тактической карты

════════════════════════════════════════
ТВОЯ ЗАДАЧА
════════════════════════════════════════

Реализовывать тактический режим боя строго по плану из файла:
  docs/bitva.md

Файл содержит 20 этапов. Каждый запланированный запуск = ОДИН этап.

ШАГ 1 — ОПРЕДЕЛИ ТЕКУЩИЙ ЭТАП:
  Прочитай docs/bitva.md
  Проверь какие файлы уже существуют:
    ls engine/tactical_battle.js ui/tactical_map.js 2>/dev/null
  Проверь git log --oneline -10 чтобы увидеть что уже сделано
  Определи следующий незавершённый этап (1–20)

ШАГ 2 — ПЕРЕД ЛЮБЫМИ ИЗМЕНЕНИЯМИ:
  Прочитай ВСЕ файлы которые будешь менять (Read tool)
  Если в bitva.md написано изменить engine/armies.js —
    сначала прочитай engine/armies.js полностью
  Если не уверен как именно интегрировать — СПРОСИ, не делай наугад

ШАГ 3 — РЕАЛИЗУЙ ТОЛЬКО ОДИН ЭТАП:
  Реализуй ровно один этап из bitva.md
  Не перепрыгивай вперёд
  Если этап большой — реализуй его полностью, не бросай на полпути

ШАГ 4 — ТЕСТИРОВАНИЕ (ОБЯЗАТЕЛЬНО):
  После реализации пройди ВСЕ чеклисты из своего этапа
  Способы тестирования:
    - Запусти игру через HTTP-сервер (python3 -m http.server 8080)
    - Проверь консоль браузера на ошибки
    - Для логических тестов — напиши и запусти node-скрипт
    - Для Canvas-тестов — открой страницу и вызови функцию вручную в консоли
  ЕСЛИ НАЙДЕНА ОШИБКА — исправь её ДО завершения сессии
  Не завершай сессию с неисправленными ошибками

ШАГ 5 — КОММИТ И ПУШ:
  git add engine/tactical_battle.js ui/tactical_map.js index.html docs/
  git commit -m "tactical: этап N — [краткое описание]"
  git push -u origin claude/historical-population-distribution-euy06

════════════════════════════════════════
ПРАВИЛА РАБОТЫ С КОДОМ
════════════════════════════════════════

1. ВСЕГДА читай файл перед изменением (Read tool)
2. НЕ переписывай функции которые работают — только добавляй новые
3. НЕ трогай engine/turn.js, engine/economy.js, engine/diplomacy.js
   если это не требует конкретный этап
4. НЕ меняй существующий resolveBattle() — только оборачивай его
5. Функции-заглушки для будущих этапов создавай пустыми:
   function futureFunction() { /* TODO: этап N */ }
6. Если код из bitva.md конфликтует с реальным кодом —
   адаптируй под реальную структуру, не ломай существующее
7. СПРАШИВАЙ если:
   - Не уверен в структуре GAME_STATE
   - Не понимаешь где именно вызывается resolveBattle()
   - Видишь что существующий код работает иначе чем описано в bitva.md
