# QA-агент Ancient Strategy — запускать каждый час

Ты — автоматический QA-агент для игры Ancient Strategy (igra2).
Рабочая ветка: `claude/historical-population-distribution-euy06`
Рабочая директория: `/home/user/igra2`

## ЗАДАЧА
Запусти игру в Playwright-песочнице, сыграй 5 ходов, кликни все
основные панели UI, выяви баги и исправь их в коде.

## ПРИНЦИП (КРИТИЧЕСКИ ВАЖНО)
Когда тест падает — сначала определи: это баг в ИГРЕ или баг в ТЕСТЕ?
- Если игра работает неправильно → исправляй КОД ИГРЫ
- Если тест проверяет неверно → исправляй ТЕСТ
- НЕЛЬЗЯ подгонять тест под баг: нельзя ставить `.catch(() => {})`,
  менять expect или пропускать проверку — нужно устранить первопричину в игре

## ШАГ 1 — Подготовка
1. `git pull origin claude/historical-population-distribution-euy06`
2. Убедись что HTTP-сервер запущен: `python3 -m http.server 8181 &`
3. Playwright браузер доступен в `/opt/node22/lib/node_modules/playwright`

## ШАГ 2 — Запуск тестов
```
node test_ui.mjs
```
Сохрани вывод и скриншоты из `/tmp/igra2_shots/`

## ШАГ 3 — Анализ результатов
Для КАЖДОЙ JS-ошибки и каждого проваленного шага:
1. Прочитай стектрейс — найди файл и строку
2. Прочитай код вокруг ошибки (Read tool)
3. Определи: это баг в игре? Да → исправляй игру
4. Запиши: `БАГИ НАЙДЕНЫ: ...` или `БАГИ НЕТ`

## ШАГ 4 — UI-обход (если игра запустилась)
Через `page.evaluate()` или клики протестируй:
- [ ] Кнопка «Следующий ход» — проходит без зависания < 15 сек
- [ ] Панель «Казна и налоги» — открывается, нет JS-ошибок
- [ ] Раздел займов — виден, кнопка «Взять заём» работает
- [ ] Слайдеры налогов — изменение отражается в превью
- [ ] Кнопка «Управление государством» — открывается
- [ ] Кнопка «Двор» — список персонажей виден
- [ ] Объявление войны соседу (если есть)
- [ ] Дипломатия — предложить торговлю

## ШАГ 5 — Исправление багов
Для каждого бага:
1. Edit нужный файл (НЕ создавай новые файлы без необходимости)
2. Запусти `test_ui.mjs` снова — убедись что баг ушёл
3. `git add` + `git commit` с описанием что исправлено
4. `git push origin claude/historical-population-distribution-euy06`

## ШАГ 6 — Повтори тест
После всех исправлений запусти `test_ui.mjs` ещё раз.
Финальный прогон должен пройти БЕЗ критических JS-ошибок.
