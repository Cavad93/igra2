## ГЛАВНЫЙ ПРИНЦИП — читай внимательно

Когда тест падает, первый вопрос: **где баг — в игре или в тесте?**

| Ситуация | Действие |
|---|---|
| Функция игры работает неправильно | Исправляй **код игры** |
| Тест проверяет несуществующий элемент | Исправляй **тест** под реальный селектор |
| Тест ловит ошибку через `.catch()` вместо исправления | Убирай заглушку, **исправляй причину** |
| JS-ошибка в консоли браузера | Читай стектрейс, **исправляй файл игры** |

**НЕЛЬЗЯ:** менять `expect`, добавлять `.catch(() => {})`, пропускать проверку,
делать тест зелёным без исправления реальной проблемы.

## ШАГ 3 — Диагностика каждой ошибки

Для **каждой** строки в выводе содержащей `⚠`, `❌`, `ERR`, `Error`, `failed`:

### 3.1 Определи тип проблемы

**Тип A — JS-ошибка в браузере** (строка начинается с адреса файла игры):
```
ui/treasury-panel.js:520 ReferenceError: nation is not defined
```
→ Читай файл `ui/treasury-panel.js` строку 520 → исправляй код игры.

**Тип B — Тест не нашёл элемент** (`selector not found`, `not visible`):
```
⚠ Раздел займов не найден
```
→ Проверь через `page.evaluate(() => document.querySelector('.tp-loans-section'))`.
Если элемент есть в DOM — исправляй селектор в тесте.
Если элемента нет — баг в игре, добавляй/исправляй рендеринг.

**Тип C — Таймаут хода** (`⏰ Ход завис`):
→ Читай `engine/turn.js`, ищи O(N²) циклы, синхронные блокировки.
Типичные причины: `Object.keys()` внутри цикла по всем нациям,
`getRelation()` создающий новые записи, бесконечный await.

**Тип D — Функция не найдена** (`showTreasuryOverlay is not a function`):
→ Проверь что скрипт загрузился: `page.evaluate(() => typeof showTreasuryOverlay)`.
Если `undefined` — проверь порядок `<script src>` в `index.html`,
нет ли синтаксической ошибки в файле которая не дала ему загрузиться.

### 3.2 Процесс исправления

1. `Read` нужный файл, изучи контекст ±20 строк вокруг ошибки
2. Найди корневую причину (не симптом)
3. `Edit` файл — минимальное исправление
4. НЕ создавай новые файлы если можно исправить существующий
